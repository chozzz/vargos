# Teach

Explain something so it sticks.

## Steps

1. Gauge the audience — what does the user already know? Ask if unclear.
2. Start with the **core idea** in 1-2 sentences — no jargon, no caveats yet
3. Build understanding:
   - **Analogy**: Connect to something familiar
   - **Concrete example**: Show it working in practice
   - **Counter-example**: Show what it's NOT or what goes wrong without it
4. Add nuance — edge cases, tradeoffs, common misconceptions
5. End with a **check**: Ask a question that tests understanding, or invite the user to try it

## Guidelines

- Lead with "what" and "why" before "how"
- One concept at a time — don't branch into related topics unless asked
- Use the user's domain for analogies when possible (check SOUL.md for context)
- Code examples should be minimal and runnable, not theoretical
- If explaining something you fetched via `web_fetch`, cite the source
